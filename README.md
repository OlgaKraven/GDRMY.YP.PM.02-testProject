# GDRMY.YP.PM.02-test-project

Тестовый учебный проект по практике для студентов СПО по модулю **ПМ.02** (сопровождение / эксплуатация информационных систем).  
Проект имитирует небольшую информационную систему для работы с театральным репертуаром и данными о показах (спектаклях) с использованием простой веб-части на **HTML + CSS** и базы данных на основе SQL-скрипта `theatre.sql`.

> Репозиторий предназначен для учебных целей: рассмотрение готового проекта, для создания своего проекта.

---

## 1. Основные возможности

В проекте реализован типичный набор страниц небольшой информационной системы:

- **Публичная часть:**
  - `index.htm` — главная страница / витрина системы (например, список спектаклей, расписание, краткое описание системы).
  - `login.htm` — форма входа пользователя (администратора / сотрудника).
  - `register.htm` — регистрация нового пользователя (при необходимости).

- **Административная часть:**
  - `admin.htm` — панель администратора (управление данными: спектакли, показы, пользователи и т.п. — в зависимости от схемы БД).
  - `reports.htm` — просмотр отчётов (например, по репертуару, показам, посещаемости, продажам и т.д.).

- **База данных:**
  - `theatre.sql` — SQL-скрипт создания и заполнения базы данных (таблицы, связи и начальные данные для тестирования).

Проект можно использовать как основу для:

- тренировки навыков **сопровождения ИС** (резервное копирование, восстановление, обновление);
- настройки **подключения к БД** и работы с данными;
- отработки типовых задач по модулю **ПМ.02**.

---

## 2. Стек технологий

В проекте используются:

- **HTML** — разметка страниц;
- **CSS** — оформление интерфейса (`/css` каталог);
- **SQL** — создание и наполнение базы данных (`theatre.sql`);
- Любой СУБД, поддерживающий синтаксис используемых команд SQL  
  (чаще всего MySQL / MariaDB — можно импортировать скрипт через phpMyAdmin или консоль).
